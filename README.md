# Notebook Application

A **full-stack Notebook / Notes application** built to demonstrate authentication, protected routes, and CRUD operations using a modern web stack.

This project allows users to **securely create, manage, and organize personal notes** after authenticating through a login system.

The goal of this project is to practice **backend architecture, authentication, and API integration** in a full-stack environment.

## Project Overview

The Notebook application consists of two main parts:

* **Backend API** – Handles authentication, user management, and note operations.
* **Frontend Client** – Provides the user interface for interacting with notes.

The backend follows a **clean MVC architecture** and exposes RESTful APIs used by the frontend.

---

## Features

### Authentication

* User Signup
* User Login
* JWT-based authentication
* Protected API routes

### Notes Management

* Create notes
* View notes
* Update notes
* Delete notes
* User-specific notes storage

### Application Features

* Secure API communication
* Organized backend structure
* Separation of concerns using MVC

## Architecture

The backend is structured using an **MVC pattern** to keep the code organized and scalable.

## Learning Focus

This project focuses on practicing:

* Backend architecture using **MVC**
* **Authentication with JWT**
* Building **RESTful APIs**
* Secure password handling with **bcrypt**
* Integrating a frontend client with a backend API
* Managing user-specific data in MongoDB


## Project Purpose

This project was built as a **learning and portfolio project** to explore how full-stack applications handle authentication and data management.

It demonstrates the fundamentals required to build scalable web applications using the **MERN ecosystem**.
